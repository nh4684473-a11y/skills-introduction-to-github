Introduction to GitHub

Welcome to GitHub!

GitHub is a collaboration platform where developers store, share, and work together on code. Learn the core concepts below to get started.

📁

Repository

A repository (or "repo") is a project folder that contains all of your files and their revision history. It is the foundation of every GitHub project.

🌿

Branch

A branch is a parallel version of your repository. You can work freely on a branch without affecting the default main branch, then merge your changes when they're ready.

💾

Commit

A commit records a snapshot of your changes. Each commit has a unique ID and a message that describes what was changed, creating a clear history of your project.

🔀

Pull Request

A pull request (PR) lets you propose changes from your branch and ask collaborators to review them before merging into the main codebase.

Your first contribution in 4 steps

  1. Create a branch

    Branch off from main to start working on your changes in isolation.

  2. Commit a file

    Add or edit a file on your branch and commit the changes with a descriptive message.

  3. Open a pull request

    Propose your changes to the team and request a review before merging.

  4. Merge your pull request

    Once approved, merge your branch into main to make your changes official.
